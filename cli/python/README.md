# tripwire-cli

Command-line client for [Tripwire](https://tripwire.so) canaries. Installs a
single `tripwire` command.

## Install

```bash
uv tool install --from . tripwire-cli
```

Or run without installing:

```bash
uvx --from . tripwire --help
```

## Usage

```bash
# Log in and cache a token. Passwordless email-code login: it prompts for your
# email (defaulting to the address from your last login) and the 6-digit code
# emailed to you.
tripwire login

# See identity, counts, and any canaries that have fired (fired first).
tripwire status

# Create a canary. The credential is shown ONCE, at creation, so capture it now.
# TYPE is a dotted id (run `tripwire canary create --help` for the full list).
tripwire canary create aws.access_key --note "warehouse reporting key"

# Write the credential to a file (mode 0600) instead of stdout.
tripwire canary create aws.access_key -o ./cred.json

# Inspect what you own (summaries only; the credential is never shown again).
tripwire canary list
tripwire canary list --type github.token --fired
tripwire canary show can_1234abcd

# Delete one.
tripwire canary delete can_1234abcd

# Download a bait bundle. With no id, a fresh bundle is issued for you first,
# then downloaded and extracted into ./<name>/.
tripwire bundle download
tripwire bundle download <id> -o ./kit    # extract into ./kit
tripwire bundle download <id> --zip       # keep the raw .zip archive
```

Create prints only the credential (or, for the `aws.profile` / `aws.credentials`
placements, a ready-to-paste config block). `list`, `show`, and `status` print
human rows by default and accept `--json` for verbatim server JSON (pipe to
`jq`). Progress and other reporting go to stderr, so stdout stays machine-clean.

Creatable types are the dotted ids `aws.access_key`, `github.token`,
`database.credentials`, `web.login`, `web.cookie`, and `k8s.config`, plus the
AWS placements `aws.profile` and `aws.credentials`. The placement's
profile/label name is generated by the backend; you cannot pass one.

## Server

`login` talks to `https://tripwire.so/api/v1` by default. Set `TRIPWIRE_SERVER`
to point at a self-hosted or test server before logging in; the server is bound
to your token at login time. The token is cached at
`~/.config/tripwire/credentials.json` (honoring `XDG_CONFIG_HOME`).
